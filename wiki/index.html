Orientierung

Warum diese Seite?

Diese Wiki-Seite soll einen kompakten, verständlichen Einstieg in das Thema Vibecoding geben. Sie richtet sich an Menschen, die mit KI schnell erste digitale Lösungen, Prototypen oder Ideen sichtbar machen möchten, ohne direkt tief in klassische Softwareentwicklung einsteigen zu müssen.

Was ist Vibecoding?

Vibecoding beschreibt das Erstellen von Software mithilfe von KI, bei dem der Fokus auf schnellen Ergebnissen, Ausprobieren und iterativer Verbesserung liegt. Statt jede Zeile Code selbst zu schreiben, beschreibt man einer KI in natürlicher Sprache, was entstehen soll, testet das Ergebnis und entwickelt es Schritt für Schritt weiter. Der Mehrwert liegt vor allem darin, Ideen schnell greifbar zu machen.

Was ist es nicht?

Vibecoding ist kein Ersatz für klassische Softwareentwicklung in komplexen oder produktionskritischen Systemen. Es ist kein Garant für sicheren, fehlerfreien oder wartbaren Code. Außerdem ist es kein Ansatz nach dem Motto „einmal prompten und fertig". Gute Ergebnisse entstehen fast immer durch mehrere Schleifen aus Testen, Prüfen und Nachschärfen.

Wann ist es sinnvoll?

Vibecoding eignet sich besonders gut für Prototyping, Lernen, kleine Tools und kreative Experimente. Es ist stark, wenn es darum geht, Ideen schnell sichtbar zu machen, Anforderungen anschaulich zu testen oder erste Klickdummys für Gespräche mit Stakeholdern zu bauen. Gerade in frühen Phasen hilft es, abstrakte Gedanken in erlebbare Entwürfe zu übersetzen.

Tipp

Vibecoding ist besonders wertvoll in Workshops: Teilnehmer sehen in Minuten erste Ergebnisse und bleiben motiviert.

Mental Model

Wie man beim Vibecoding denkt

Vibecoding funktioniert am besten, wenn man nicht in fertigen Endlösungen denkt, sondern in Lern- und Verbesserungszyklen. Ziel ist nicht Perfektion im ersten Schritt, sondern ein schneller erster Entwurf, der anschließend gemeinsam mit der KI verfeinert wird.

Iterativer Workflow

Idee → Prompt → Output → Test → Feedback → Iteration

Zuerst wird eine Idee beschrieben. Daraus erzeugt die KI einen ersten Output, zum Beispiel eine kleine Anwendung oder einen Prototypen. Dieser wird getestet, bewertet und mit Feedback verbessert. Dieser Zyklus wiederholt sich, bis das Ergebnis nützlich und stimmig ist.

Rollenmodell

Rolle Aufgabe
Human Ziel definieren, Kontext geben, Qualität bewerten
AI Code generieren, Vorschläge machen, Varianten liefern
Human Testen, steuern, priorisieren, entscheiden

Die KI übernimmt nicht die Verantwortung für das Ergebnis. Sie ist eher wie ein sehr schneller Umsetzer, der Unterstützung liefert. Die fachliche Richtung, die Bewertung und die Entscheidung bleiben beim Menschen.

Setup

Werkzeuge

Für den Einstieg braucht es meist weniger Infrastruktur, als viele vermuten. Schon mit Browser, KI-Tool und einem Ort zum Speichern oder Veröffentlichen lassen sich erste Ergebnisse bauen und zeigen.

Minimal-Setup

Ein sinnvolles Minimal-Setup besteht aus:

  • einem Browser,
  • einem KI-Tool wie ChatGPT, Claude oder Copilot,
  • einem Ort zur Ablage des Codes, z. B. lokal oder auf GitHub.

Damit lassen sich einfache Web-Prototypen oft schon vollständig erstellen, testen und teilen.

Browser

Der Browser ist die wichtigste Testumgebung für einfache Web-Projekte. HTML-, CSS- und JavaScript-Prototypen lassen sich direkt lokal öffnen und sofort ausprobieren. Das senkt die Einstiegshürde erheblich.

ChatGPT / Copilot / Claude

Verschiedene Tools eignen sich für verschiedene Phasen. ChatGPT ist stark bei Strukturierung, Brainstorming und freiem Dialog. Claude Code eignet sich gut für technische Umsetzung und Datei-Arbeit. Copilot unterstützt eher direkt in der Entwicklungsumgebung beim Schreiben und Ergänzen von Code.

GitHub

GitHub hilft dabei, Projekte sauber abzulegen, Versionen zu speichern und Ergebnisse später zu veröffentlichen. Für einfache statische Projekte ist GitHub außerdem ein guter Ausgangspunkt, um mit GitHub Pages kostenlos eine Website bereitzustellen.

Deployment Flow

Ein typischer einfacher Veröffentlichungsweg:

  1. Repository erstellen
  2. HTML-Datei hinzufügen
  3. GitHub Pages aktivieren
  4. URL teilen

Damit kann ein Workshop-Ergebnis sehr schnell aus einem lokalen Entwurf zu einer öffentlich erreichbaren Seite werden.

Erste Erfolge

Quick Wins

Gerade am Anfang sind kleine, überschaubare Projekte ideal. Sie liefern schnell sichtbare Ergebnisse und helfen dabei, ein Gefühl für die Zusammenarbeit mit KI zu entwickeln.

Mini-Projekte

Geeignet sind besonders kleine Projekte mit klarer Oberfläche und einfacher Logik. Sie sind schnell testbar und überschaubar genug, um Iteration und Feedback direkt zu erleben.

To-Do App

Eine To-Do-App ist ein guter Einstieg, weil sie viele typische UI-Elemente enthält: Eingaben, Listen, Buttons und einfache Zustände. Gleichzeitig bleibt die fachliche Logik leicht verständlich.

Taschenrechner

Ein Taschenrechner eignet sich gut, um einfache Eingabelogik, Zustandsänderungen und Fehlerfälle zu verstehen. Er ist klein genug, um schnell fertig zu werden, aber komplex genug für erste Lernmomente.

Mini-Spiel

Ein kleines Spiel ist besonders motivierend, weil Ergebnisse sofort sichtbar sind. Außerdem zeigt ein Mini-Spiel gut, wie man mit KI kreative Ideen, einfache Logik und Interaktion kombinieren kann.

Beispiel

„Erstelle ein Pong-Spiel mit zwei Schlägern und Punktestand. Einfaches Design, spielbar im Browser." – Solche Prompts reichen oft schon für einen funktionierenden Prototyp.

Systematisches Bauen

Vom schnellen Prototyp zur Struktur

Sobald erste Ergebnisse funktionieren, lohnt sich ein systematischerer Blick auf Aufbau und Bestandteile. Das hilft, Änderungen sauberer umzusetzen und spätere Erweiterungen zu erleichtern.

UI (HTML/CSS)

Die Benutzeroberfläche beschreibt, wie die Anwendung aussieht. HTML legt die Struktur fest, CSS das Design. Im Vibecoding kann die KI sehr schnell erste Layouts, Formulare, Navigationen oder Kartenansichten erzeugen, die dann schrittweise angepasst werden.

Logik (JavaScript)

JavaScript steuert das Verhalten einer Web-Anwendung. Dazu gehören Klicks, Berechnungen, Zustandswechsel oder Validierungen. Gerade hier zeigt sich, warum Testen wichtig ist: Die Oberfläche kann gut aussehen, obwohl die Logik im Hintergrund noch fehlerhaft ist.

State Management

Auch kleine Anwendungen haben Zustände: Welche Aufgabe ist erledigt? Welcher Filter ist aktiv? Welcher Nutzerpfad wurde gewählt? Wer Vibecoding länger nutzt, merkt schnell, dass ein gutes Verständnis von Zuständen hilft, Änderungen nachvollziehbar und stabil zu machen.

APIs einbinden

Viele spannende Prototypen werden erst dann besonders nützlich, wenn sie externe Daten einbinden. APIs ermöglichen zum Beispiel Wetterdaten, Suchfunktionen oder Datenabfragen. Hier ist besondere Vorsicht nötig, weil externe Schnittstellen Fehlerquellen, Limits und Sicherheitsfragen mitbringen.

Achtung

API-Keys niemals direkt im Frontend-Code hinterlegen! Öffentlich sichtbare Seiten machen solche Schlüssel für jeden zugänglich.

Qualität und Risiken

Warum dieser Abschnitt wichtig ist

KI kann schnell Ergebnisse liefern, aber nicht automatisch gute Ergebnisse. Deshalb gehört zur Arbeit mit Vibecoding immer auch eine realistische Einordnung der Qualität.

Typische Probleme

Gerade bei schnellen Prototypen treten oft typische Schwächen auf: unklare Logik, widersprüchliche Funktionen, brüchige UI, unsaubere Struktur oder fehlende Randfallbehandlung. Diese Probleme sind normal, solange man sie erkennt und systematisch bearbeitet.

Halluzinationen

KI kann Dinge behaupten oder erzeugen, die plausibel wirken, aber sachlich falsch oder technisch nicht funktionsfähig sind. Das gilt für Code genauso wie für Erklärungen. Deshalb sollten Ergebnisse nie ungeprüft übernommen werden.

Sicherheitslücken

KI-generierter Code ist nicht automatisch sicher. Besonders bei Formularen, Benutzerangaben, externen Datenquellen oder Authentifizierung können schnell Schwachstellen entstehen. Vibecoding ist deshalb ungeeignet, um sicherheitskritische Systeme ohne fachliche Prüfung zu bauen.

Schlechter Code

Auch wenn etwas funktioniert, kann der Code schwer verständlich, schlecht strukturiert oder schwer wartbar sein. Gerade bei mehreren Iterationen entstehen schnell doppelte Logik, unnötige Komplexität oder unklare Abhängigkeiten.

Debugging mit KI

Ein praktikabler Ansatz ist, Fehler gezielt mit der KI zu debuggen. Statt nur „es geht nicht" zu schreiben, hilft es, Fehlermeldungen, beobachtetes Verhalten und den betroffenen Codeabschnitt mitzugeben. So kann die KI präziser helfen.

Fehler kopieren

Besonders effektiv ist es, konkrete Fehlermeldungen oder problematische Stellen exakt zu kopieren. So vermeidet man Missverständnisse und erhöht die Chance auf eine passende Lösung.

Gezielt nach Fix fragen

Gute Debugging-Prompts sind konkret. Hilfreich ist zum Beispiel:

Dieser Button reagiert nicht. Hier ist der relevante Code und die Fehlermeldung. Bitte finde die Ursache und schlage eine minimale Korrektur vor.

So bekommt man meist bessere Ergebnisse als mit rein allgemeinen Fragen.

Stolperfalle

KI-generierter Code kann funktionierend aussehen, obwohl er grundlegende Logikfehler enthält. Immer manuell testen!

Deployment und Sharing

Warum Veröffentlichen wichtig ist

Ein Prototyp wird deutlich wertvoller, wenn andere ihn sehen, testen oder kommentieren können. Deployment macht aus einer lokalen Idee ein teilbares Ergebnis.

GitHub Pages

GitHub Pages ist für einfache Websites ein besonders leichter Weg, Projekte kostenlos zu veröffentlichen. Man braucht nur ein Repository, eine HTML-Datei und die Aktivierung der Pages-Funktion. Danach ist das Ergebnis über eine URL erreichbar.

Versionierung

Versionierung hilft dabei, Änderungen nachvollziehbar zu halten. Statt nur den neuesten Stand zu kennen, kann man frühere Versionen sichern, Unterschiede vergleichen und Fehler leichter zurückverfolgen. Gerade bei iterativer Arbeit mit KI ist das sehr hilfreich.

Teilen von Links

Ein veröffentlichter Link ist oft die einfachste Form, einen Prototypen im Team oder Workshop zu teilen. Das erleichtert Feedback, Diskussionen und gemeinsames Lernen erheblich.

Projekte (Sandbox)

Freies Arbeiten und Experimentieren

Die Sandbox ist der Raum für Ausprobieren. Hier geht es nicht um Perfektion, sondern um Geschwindigkeit, Kreativität und Lernen. Kleine Ideen dürfen hier unfertig, verspielt oder experimentell sein. Gerade das freie Arbeiten sorgt oft für die stärksten Aha-Momente.

Tipp

Vibe-Coding ist kein Hexenwerk. Wer eine Idee beschreiben kann, kann auch erste Ergebnisse erzeugen.

Golden Path Tutorial

Ein sinnvoller Standardablauf für Einsteiger:

Idee definieren

Am Anfang steht kein perfektes Fachkonzept, sondern eine verständliche Idee. Wichtig ist nur, dass das Ziel klar genug beschrieben werden kann.

Prompt formulieren

Danach wird die Idee in eine konkrete Anweisung für die KI übersetzt. Gute Prompts beschreiben Ziel, Nutzer, Funktionsumfang und gewünschtes Verhalten möglichst klar.

Code generieren

Die KI erstellt daraufhin einen ersten Entwurf. Das kann ein einfacher Screen, eine kleine Anwendung oder ein klickbarer Prototyp sein.

Testen

Danach wird ausprobiert, was schon funktioniert und was noch fehlt. Erst im Test zeigt sich oft, wo Logik, Nutzerführung oder Details noch unklar sind.

Debuggen

Fehler sind Teil des Prozesses. Wichtiger als Fehlerfreiheit im ersten Anlauf ist die Fähigkeit, Probleme systematisch zu erkennen und mit der KI daran weiterzuarbeiten.

Deployen

Zum Abschluss kann das Ergebnis lokal genutzt oder online veröffentlicht werden, etwa über GitHub Pages oder ein einfaches Hosting-Setup.

Anti-Vibecoding

Wofür Vibecoding nicht geeignet ist

Vibecoding ist nicht geeignet für sicherheitskritische Systeme, große Anwendungen mit komplexer Architektur oder produktive Unternehmenslösungen ohne fachliche und technische Absicherung. Sobald Themen wie Datenschutz, Compliance, Betrieb, Integration oder Skalierung zentral werden, reicht reines Vibecoding nicht aus.

Achtung

Vibecoding ist ein Werkzeug für Prototypen, Lernen und kreative Experimente – kein Ersatz für professionelle Softwareentwicklung in kritischen Kontexten.

Code Reading Light

Warum minimales Codeverständnis hilft

Man muss kein Entwickler sein, um mit Vibecoding gute Ergebnisse zu erzielen. Trotzdem hilft es, einfache Strukturen im Code grob lesen und nachvollziehen zu können.

Variablen erkennen

Variablen speichern Werte und Zustände. Wer erkennt, welche Informationen wo gehalten werden, versteht schneller, warum sich eine Anwendung auf bestimmte Weise verhält.

Funktionen verstehen

Funktionen bündeln Verhalten. Schon ein grobes Verständnis davon, welche Funktion was auslöst, hilft beim Testen und Formulieren von Änderungswünschen.

Änderungen nachvollziehen

Bei iterativer Arbeit mit KI ist es wichtig zu sehen, was sich von Version zu Version verändert hat. So erkennt man besser, ob ein Fix wirklich geholfen oder an anderer Stelle etwas verschlechtert hat.

Refactoring mit KI

Besseren Code erzeugen

KI kann nicht nur neuen Code erzeugen, sondern auch bestehenden Code verbessern. Das ist besonders hilfreich, wenn ein Prototyp zwar funktioniert, aber unübersichtlich geworden ist.

„Optimiere meinen Code"

Ein nützlicher Anwendungsfall ist, die KI um Vereinfachung oder bessere Struktur zu bitten. Dabei sollte man möglichst klar sagen, was verbessert werden soll: Lesbarkeit, Kürze, Wiederverwendbarkeit oder Trennung von Logik und Darstellung.

„Mache ihn wartbarer"

Wartbarkeit bedeutet, dass Änderungen später leichter möglich sind. Die KI kann dabei helfen, doppelte Logik zu reduzieren, Funktionen sauberer aufzuteilen oder Benennungen verständlicher zu machen.

Beispiel-Prompt

„Dieser Code funktioniert, ist aber unübersichtlich. Bitte vereinfache ihn, ohne die Funktionalität zu ändern. Trenne UI und Logik."

Kosten und Limits

Warum das relevant ist

Auch wenn viele erste Schritte günstig oder kostenlos möglich sind, stoßen KI-Tools und externe Dienste an Grenzen.

API-Kosten beachten

Wer mit APIs oder kostenpflichtigen KI-Modellen arbeitet, sollte die Nutzung im Blick behalten. Gerade bei wiederholten Anfragen oder größeren Projekten können Kosten entstehen, die anfangs unterschätzt werden.

Token Limits

KI-Modelle haben Begrenzungen bei der Menge an Eingabe- und Ausgabedaten. Sehr große Kontexte, lange Codeblöcke oder umfangreiche Verläufe können dazu führen, dass Informationen abgeschnitten oder schlechter verarbeitet werden.

Rate Limits

Einige Dienste begrenzen, wie oft man in kurzer Zeit Anfragen senden darf. Das ist besonders relevant, wenn Prototypen externe APIs nutzen oder automatisiert viele Anfragen stellen.

Tipp

Viele KI-Tools bieten kostenlose Stufen an, die für Workshop-Projekte völlig ausreichen. Bezahlte Modelle lohnen sich erst bei regelmäßiger, intensiver Nutzung.

Vom Playground zum Produkt

Der Übergang in die Realität

Ein funktionierender Prototyp ist noch kein Produkt. Wer aus einer Idee mehr machen will, muss zusätzliche Themen berücksichtigen.

Hosting

Für erste Schritte reichen einfache Hosting-Lösungen wie GitHub Pages. Später können Self-Hosting, Docker oder professionellere Plattformen sinnvoll werden.

Nutzerfeedback

Sobald andere Menschen ein Tool nutzen, wird ihr Feedback zur wichtigsten Weiterentwicklungsquelle. Häufig zeigt erst echte Nutzung, welche Funktionen wirklich hilfreich sind.

Iteration

Der Weg vom Playground zum Produkt ist fast immer ein Prozess aus Nutzung, Feedback, Anpassung und erneuter Prüfung. Vibecoding kann diesen Weg beschleunigen, ersetzt aber nicht die saubere Produktarbeit.

Prompt-Bibliothek

Warum gute Prompts wichtig sind

Die Qualität der Ergebnisse hängt stark davon ab, wie klar Ziel, Kontext und Erwartung beschrieben werden. Gute Prompts sparen Iterationen und führen schneller zu brauchbaren Ergebnissen.

Gute Prompts

Gute Prompts benennen: das Ziel, die Zielgruppe, die gewünschte Funktion, den Umfang, Einschränkungen, und idealerweise Beispiele oder Referenzen.

Erstelle eine einfache To-Do-Web-App für private Aufgaben. Links Kategorien, rechts Aufgabenliste. Jede Aufgabe hat Checkbox, Titel und optionales Datum. Bitte schlichtes Layout, mobil nutzbar, ohne Login und ohne Prioritäten.

Schlechte Prompts

Schlechte Prompts sind meist zu kurz, zu vage oder widersprüchlich. Ein Prompt wie „Bau mir eine coole App" lässt zu viel Interpretationsspielraum und führt selten direkt zu einem guten Ergebnis.

Vorlagen

Es lohnt sich, wiederverwendbare Prompt-Vorlagen zu sammeln, etwa für: Ideenstrukturierung, UI-Entwürfe, Debugging, Refactoring oder das Erzeugen erster MVPs.

Common Failures

UI kaputt

Ein häufiges Problem ist eine Oberfläche, die zwar grundsätzlich sichtbar ist, aber Layoutfehler, unklare Abstände oder schlechtes Verhalten auf kleineren Displays hat. Hier hilft es, gezielt auf einzelne Stellen hinzuweisen.

Stolperfalle

Immer auf dem Handy testen! Viele KI-generierte Layouts sehen am Desktop gut aus, brechen aber auf kleinen Bildschirmen.

Code funktioniert nicht

Manchmal erzeugt die KI formal plausiblen Code, der in der Praxis Fehler wirft. Dann ist sauberes Debugging wichtig: Fehlermeldung kopieren, Ursache eingrenzen, minimalen Fix anfordern.

Logikfehler

Besonders tückisch sind Situationen, in denen die Anwendung funktioniert, aber inhaltlich das Falsche tut. Solche Fehler entdeckt man nur durch echtes Testen aus Nutzersicht.

Workflow Visualisierung

Der Standardzyklus

Idee → Prompt → Output → Test → Feedback → Iteration

Diese einfache Schleife ist das Kernmodell des Vibecodings. Wer sie verinnerlicht, arbeitet produktiver mit KI und erwartet nicht, dass perfekte Ergebnisse schon beim ersten Versuch entstehen.

Hosting-Anleitung

Schritt-für-Schritt-Anleitung: Vom lokalen Ergebnis bis zur eigenen Website.
Zielgruppe: Workshop-Teilnehmer ohne Vorkenntnisse.

Diese Anleitung zeigt drei Wege, wie du dein Vibe-Coding-Projekt nutzen und zugänglich machen kannst - vom einfachsten Einstieg bis hin zu fortgeschrittenen Optionen.

Lokale Ergebnisse mit Claude Code

Der einfachste Weg, um mit Vibe-Coding loszulegen: Claude Code liefert dir die fertigen Dateien direkt in einen Ordner auf deinem Computer.

So funktioniert es:

  1. Ordner anlegen - erstelle auf deinem Desktop (oder an einem beliebigen Ort) einen neuen Ordner, z. B. mein-projekt
  2. Claude Code öffnen - starte Claude Code und wähle diesen Ordner als Arbeitsverzeichnis
  3. Idee beschreiben - sage Claude Code, was du bauen möchtest. Die KI erstellt die nötigen Dateien (HTML, CSS, JavaScript) direkt in deinem Ordner
  4. Im Browser öffnen - öffne die Datei index.html aus deinem Ordner per Doppelklick oder über Datei > Öffnen in deinem Browser

Das ist der schnellste Weg, um deine Ergebnisse zu sehen. Du brauchst keinen Server, kein Konto und keine zusätzliche Software - nur Claude Code und einen Browser.

Dein Projekt liegt dabei nur auf deinem eigenen Computer. Wenn du es mit anderen teilen oder im Internet zugänglich machen möchtest, lies weiter.

Für Fortgeschrittene: GitHub Pages

Was ist GitHub?

GitHub ist eine Plattform im Internet, auf der man Code-Projekte speichern und mit anderen teilen kann - ähnlich wie eine Cloud-Ablage, aber speziell für Quellcode. Du brauchst dafür ein kostenloses Konto unter github.com.

Claude Code kann direkt mit GitHub verbunden werden. So werden deine Dateien nicht nur lokal gespeichert, sondern auch auf GitHub hochgeladen. Für kleine, öffentlich zugängliche Projekte bietet GitHub mit GitHub Pages sogar kostenloses Hosting - deine Seite bekommt eine eigene Internetadresse.

Schritt-für-Schritt-Anleitung

  1. GitHub-Konto erstellen - registriere dich kostenlos auf github.com
  2. Repository erstellen - lege ein neues Projekt (Repository) auf GitHub an
  3. Claude Code mit GitHub verbinden - Claude Code kann deine Dateien direkt in das Repository hochladen
  4. Pages aktivieren - im Repository unter Settings > Pages den Branch main und Ordner / (root) auswählen, dann speichern
  5. Fertig - die Seite ist nach ca. 1-2 Minuten unter https://<benutzername>.github.io/<repository-name>/ erreichbar

Änderungen, die auf main hochgeladen werden, werden automatisch veröffentlicht.

Hinweis: Basispfade

GitHub Pages liefert die Seite unter einem Unterpfad aus (z. B. /vibecoding-academy/). Wenn du die Seite lokal öffnest, liegt sie dagegen direkt unter /. Relative Links müssen ggf. angepasst werden.

Self-Hosting mit Docker

Was ist Docker?

Docker ist ein Werkzeug, das eine Anwendung mitsamt allem, was sie braucht, in einen tragbaren "Container" verpackt. Stell dir das wie einen kleinen, eigenständigen Mini-Computer vor, der deine Website ausliefert. Du installierst Docker einmal auf deinem Rechner, und danach reicht ein einziger Befehl, um deine Seite zu starten.

Docker eignet sich, wenn du deine Seite auf einem eigenen Server betreiben oder lokal in einer professionellen Umgebung testen möchtest.

Voraussetzung

Docker muss installiert sein: https://docs.docker.com/get-docker/

Variante A: Mit Dockerfile (empfohlen für eigene Images)

1. Dockerfile im Projektordner erstellen:

FROM nginx:alpine
COPY . /usr/share/nginx/html
EXPOSE 80

2. Image bauen und starten:

docker build -t vibecoding-academy .
docker run -d -p 8080:80 vibecoding-academy

3. Im Browser öffnen: http://localhost:8080

4. Container stoppen:

docker ps                        # Container-ID nachschauen
docker stop <container-id>       # Container stoppen

Variante B: Ohne Dockerfile (schnelle lokale Vorschau)

Im Projektordner ausführen:

docker run -d -p 8080:80 -v $(pwd):/usr/share/nginx/html:ro nginx:alpine

Dieser Befehl startet einen nginx-Container und bindet den aktuellen Ordner direkt ein - ohne etwas zu bauen.
