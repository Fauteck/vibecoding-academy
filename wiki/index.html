Worum es in diesem Workshop geht

Die Vibe-Coding Academy soll zeigen, was mit KI-gestützter Entwicklung heute schon möglich ist -- und wie niedrig die Einstiegshürde dabei sein kann.

Es geht nicht darum, in kurzer Zeit ein vollständig ausgereiftes, produktiv einsetzbares Unternehmens-Tool zu bauen. Stattdessen geht es um:

  • Neugier und Ausprobieren
  • Kreativität bei der Ideenentwicklung
  • schnelles Prototyping
  • iteratives Verbessern
  • ein besseres Verständnis dafür, wie man mit KI zusammenarbeitet

Vibe-Coding ist kein Hexenwerk. Wer eine Idee beschreiben kann, kann auch erste Ergebnisse erzeugen.

Was Vibe-Coding ausmacht

Vibe-Coding bedeutet, mit einer KI gemeinsam Anwendungen, Prototypen oder Klickdummys zu entwickeln -- oft in natürlicher Sprache statt über klassischen Code von Grund auf.

Der Schlüssel ist dabei nicht nur Technik, sondern vor allem:

  • Kreativität
  • klare Kommunikation
  • Bereitschaft zur Iteration

Man muss also nicht zwingend selbst programmieren können, um etwas zu erschaffen. Hilfreich ist es aber, wenn man:

  • digitale Infrastruktur grob versteht,
  • bestehende Logiken nachvollziehen kann,
  • und bei Bedarf einfachen Code zumindest ansatzweise lesen oder prüfen kann.

Die wichtigste Erkenntnis

Man braucht keine Coding-Erfahrung, um mit Vibe-Coding zu guten ersten Ergebnissen zu kommen.

Was man allerdings braucht:

  • eine Idee,
  • die Fähigkeit, diese Idee zu beschreiben,
  • und die Bereitschaft, der KI Rückmeldung zu geben.

Zusammenarbeit mit KI: wie mit einem Werkstudenten

Ein hilfreiches Bild ist: Arbeite mit der KI wie mit einem Werkstudenten oder einer Werkstudentin.

Das bedeutet:

  • Anforderungen möglichst klar formulieren
  • Kontext mitgeben
  • Erwartungen konkret benennen
  • Rückfragen zulassen
  • Ergebnisse prüfen und nachschärfen

Je klarer die Aufgabe beschrieben ist, desto besser wird das Ergebnis.

Gleichzeitig darf man nicht erwarten, dass die KI von Anfang an alles perfekt versteht. Gute Ergebnisse entstehen oft im Dialog.

Iterativ statt perfekt von Anfang an

Anders als im klassischen Projekt-, Produkt- oder Anforderungsmanagement muss beim Vibe-Coding nicht jeder Schritt von Anfang an vollständig durchpriorisiert und bis ins letzte Detail ausdefiniert sein.

Oft ist es sinnvoller, mit einem ersten Aufschlag zu starten:

  1. Idee grob beschreiben
  2. ersten Prototyp oder ersten Entwurf erzeugen lassen
  3. anschauen, bewerten, testen
  4. Rückmeldung geben
  5. iterativ verbessern

Typische Rückmeldungen an die KI sind zum Beispiel:

  • „Den Button wollte ich anders haben.“
  • „Das Dropdown soll sich anders verhalten.“
  • „Hier fehlt noch ein Schritt.“
  • „Der Ablauf ist noch nicht intuitiv genug.“
  • „Bitte ergänze noch einen alternativen Nutzerpfad.“

Gerade diese iterative Arbeitsweise ist eine der größten Stärken von Vibe-Coding.

Paradigmenwechsel: Bauen, was DU brauchst

Vibe-Coding verändert nicht nur wie Software entsteht, sondern auch für wen.

Bisher: Entwicklung für die Mehrheit

In der klassischen Softwareentwicklung baut ein Entwicklerteam Features für möglichst viele Nutzer:

  • Anforderungen werden über Umfragen und Nutzerstudien erhoben
  • Features werden auf Nachfrage priorisiert
  • Das Team entscheidet nach eigener Vorstellung, wie etwas genutzt wird
  • Nischenwünsche einzelner Nutzer fallen oft hinten runter

Neu: Entwicklung für DICH

Mit Vibe-Coding kannst du dir ein Tool genau so bauen, wie du es brauchst:

  • Du entscheidest, welche Funktionen rein kommen -- und welche nicht
  • Du kannst Features einbauen, die außer dir niemand brauchen würde
  • Du lässt alles weg, was dich nicht betrifft
  • Das Tool passt sich dir an -- nicht umgekehrt

Du bist nicht mehr Nutzer eines Standardprodukts, sondern Auftraggeber deiner eigenen Lösung.

Genau das macht Vibe-Coding so mächtig: Du musst nicht mehr darauf warten, dass jemand anderes dein Problem löst.

Was Vibe-Coding besonders gut kann

Vibe-Coding ist besonders stark, wenn es darum geht:

  • Ideen schnell sichtbar zu machen
  • theoretische Anforderungen in Prototypen zu übersetzen
  • Klickdummys für Stakeholder zu bauen
  • erste Funktionsideen erlebbar zu machen
  • Anforderungen anschaulich mit anderen abzugleichen

Ein großer Vorteil: Dinge, die in einem schriftlichen Konzept oft noch abstrakt bleiben, werden im Prototypen viel greifbarer.

Zum Beispiel lässt sich oft erst im Ausprobieren wirklich beurteilen:

  • wie sich ein Button verhalten soll,
  • wie ein Dropdown funktionieren sollte,
  • wie ein Nutzerfluss logisch wirkt,
  • oder welche Informationen an welcher Stelle fehlen.

Nutzen im Projektkontext

Ein vibe-gecodeter Prototyp kann in Projekten sehr wertvoll sein -- auch dann, wenn er nicht produktiv genutzt werden soll.

Er kann helfen als:

  • Gesprächsgrundlage mit Stakeholdern
  • visuelle Ergänzung zu Fachkonzepten
  • Diskussionsbasis für Anforderungen
  • Anschauungsmaterial für Entwicklerinnen und Entwickler

So erhalten Entwicklerteams nicht nur ein schriftliches Konzept, sondern auch eine grobe, visuelle Vorstellung davon, wie ein Feature gedacht ist und wie sich Details verhalten könnten.

Das ersetzt keine saubere technische Konzeption -- kann aber das gemeinsame Verständnis deutlich verbessern.

Was dieser Workshop ausdrücklich nicht leisten soll

Der Workshop zielt nicht darauf ab, am Ende ein produktionsreifes Tool für den Unternehmenseinsatz zu bauen.

Denn dafür müssten zusätzlich viele weitere Fragen geklärt werden, zum Beispiel:

  • Datenschutz
  • IT-Sicherheit
  • technische Architektur
  • Wartbarkeit
  • Integration in bestehende Systeme
  • Betrieb und Verantwortlichkeiten
  • Compliance und Governance

Der Fokus des Workshops liegt daher bewusst auf dem Erkunden von Möglichkeiten -- nicht auf der finalen Produktivsetzung.

Gute Haltung für den Einstieg

Wer mit Vibe-Coding starten möchte, braucht vor allem:

  • Neugier
  • Mut zum Ausprobieren
  • Kreativität
  • Offenheit für Zwischenschritte

Es reicht völlig, mit einer groben Idee zu starten und einfach zu schauen:

„Was passiert, wenn ich diese Anforderung mal an eine KI übergebe?“

Genau daraus entstehen oft die besten Aha-Momente.

Copycat-Prinzip: Gut adaptiert besser als schlecht selbst gedacht

Eine der effektivsten Strategien beim Vibe-Coding: Lass dich von bestehenden Apps und Features inspirieren.

Statt jede Funktion von Grund auf neu zu beschreiben, kannst du der KI anschaulich zeigen, was du möchtest -- indem du auf Bekanntes verweist:

  • Bestehende Apps als Referenz nutzen -- „Die Navigation soll so funktionieren wie bei Spotify“
  • Features aus anderen Tools adaptieren -- „Ich möchte eine Drag-and-Drop-Sortierung wie bei Trello“
  • Screenshots oder Beschreibungen mitgeben -- „Das Layout soll ungefähr so aussehen wie auf diesem Bild“
  • Bewusst abgrenzen -- „Ähnlich wie bei Excel, aber ohne die Formeln -- nur eine einfache Tabelle“

Ich möchte eine Aufgabenliste ähnlich wie bei Todoist: links eine Liste mit Kategorien, rechts die einzelnen Aufgaben. Jede Aufgabe hat eine Checkbox, einen Titel und ein optionales Fälligkeitsdatum. Aber ohne Prioritäten und ohne Kommentarfunktion -- halte es bewusst einfach.

Du musst das Rad nicht neu erfinden. Gute Ideen zu adaptieren und auf deine Bedürfnisse zuzuschneiden ist oft schneller und besser, als alles selbst von Grund auf zu beschreiben.

Praktischer Tipp: Idee erst strukturieren lassen

Ein sehr nützlicher Ansatz ist, nicht sofort mit dem Coding zu beginnen, sondern die Idee zunächst mit einer KI zu strukturieren.

Zum Beispiel so:

  • Idee als Sprachnachricht oder freies Brainstorming festhalten
  • einfach 5 bis 10 Minuten „drauflos erzählen“
  • die KI bitten, daraus ein strukturiertes Konzept zu machen

Dabei kann die KI helfen, indem sie:

  • Gedanken ordnet
  • Anforderungen strukturiert zusammenfasst
  • offene Punkte identifiziert
  • Rückfragen zu Missverständnissen stellt
  • fehlende Aspekte sichtbar macht
  • eigene Ideen und Ergänzungen einbringt

So entsteht eine deutlich bessere Ausgangsbasis für das eigentliche Vibe-Coding.

Beispiel für einen sinnvollen Prompt an eine KI (z. B. ChatGPT)

Ich schildere dir jetzt frei eine Idee für ein digitales Tool. Bitte strukturiere meine Gedanken, fasse die Anforderungen übersichtlich zusammen, markiere offene Punkte und mögliche Missverständnisse und stelle mir Rückfragen. Ergänze außerdem eigene Vorschläge für sinnvolle Funktionen oder Aspekte, an die ich vielleicht noch nicht gedacht habe.

KI-Kompetenzen aufteilen: Das richtige Tool für die richtige Aufgabe

Nicht jede KI ist für jede Aufgabe gleich gut geeignet. Ein bewährter Ansatz ist, verschiedene Tools gezielt für unterschiedliche Phasen einzusetzen:

Phase Empfohlenes Tool Warum
Idee strukturieren, Brainstorming, Konzept erstellen ChatGPT Stark im freien Dialog, Strukturierung von Gedanken, kreative Ergänzungen
Coding, technische Umsetzung, Implementierung Claude (Code) Spezialisiert auf Code-Generierung, versteht Projektkontext, arbeitet direkt mit Dateien
Recherche, Vergleiche, Entscheidungshilfen ChatGPT oder Claude Beide gut geeignet -- je nach persönlicher Präferenz

Empfohlener Workflow

  1. Schritt 1: Idee mit ChatGPT besprechen und strukturieren lassen
  2. Schritt 2: Das fertige Konzept als Grundlage an Claude Code übergeben
  3. Schritt 3: Claude Code setzt die technische Umsetzung um
  4. Schritt 4: Ergebnis testen, Feedback geben, iterativ verbessern

Tipp: Du kannst die Ideenphase auch als Sprachnachricht in ChatGPT starten -- einfach drauflos erzählen und die KI das Konzept daraus bauen lassen. Das fertige Konzept kopierst du dann zu Claude Code.

Erfolgsfaktoren für gutes Vibe-Coding

1. Klar formulieren

Beschreibe Ziel, Nutzergruppe, gewünschten Ablauf und Funktionsumfang möglichst konkret.

2. Nicht auf Perfektion beim ersten Versuch warten

Ein erster Entwurf darf unvollständig sein.

3. Iterativ arbeiten

Gib Feedback, korrigiere, ergänze, schärfe nach.

4. Visualisierung nutzen

Ein Prototyp zeigt oft besser als ein Text, wie etwas gemeint ist.

5. Die KI mitdenken lassen

Bitte die KI aktiv um Rückfragen, Verbesserungsvorschläge und alternative Ideen.

6. Ergebnisse realistisch einordnen

Ein guter Prototyp ist noch kein produktionsreifes Produkt.

Rahmenbedingungen festlegen: CLAUDE.md & Co.

Wenn du regelmäßig mit KI-Tools arbeitest, lohnt es sich, verbindliche Regeln für die KI in einer Datei festzulegen. So muss man nicht bei jedem Gespräch die gleichen Grundsätze wiederholen.

Was ist eine CLAUDE.md?

Eine CLAUDE.md ist eine Markdown-Datei im Repository-Root, die Claude Code automatisch als Regelbasis erkennt. Sie enthält Projektregeln, Architekturvorgaben, erlaubte und verbotene Aktionen, Qualitätsanforderungen und mehr. Andere KI-Tools nutzen ähnliche Dateien unter anderen Namen.

Kompatibilität mit verschiedenen KI-Tools

Tool Erwarteter Dateipfad
Claude Code CLAUDE.md (Repository-Root)
GitHub Copilot .github/copilot-instructions.md
Cursor .cursor/rules oder .cursorrules
Windsurf .windsurfrules
Cline .clinerules

Tipp: Erstelle eine zentrale Regeldatei und lege Symlinks oder Kopien unter den jeweiligen Pfaden an, damit alle Tools dieselben Regeln verwenden.

Beispiel-Template (anonymisiert)

Das folgende Template zeigt eine typische Struktur. Passe die Inhalte an dein Projekt an:

# CLAUDE.md — Regeln für KI-gestützte Entwicklung

## Projektübersicht
- **Typ:** [Web-App / CLI / Bibliothek / statische Website]
- **Stack:** [z. B. HTML, CSS, JavaScript / Python, Flask / etc.]
- **Deployment:** [z. B. GitHub Pages / Docker / Cloud]

## Grundprinzipien
- Produktion wird ausschließlich über versionierte Deployments aktualisiert
- Keine manuellen Hotfixes in Produktion
- Der main-Branch spiegelt stets den Live-Stand wider

## Erlaubt / Nicht erlaubt

### Die KI darf
- Code ändern und refactoren (im Rahmen bestehender Architektur)
- Dateien erstellen/ändern (inkl. Tests und Doku)
- Build-/CI-Konfiguration anpassen (wenn begründet)

### Die KI darf NICHT
- Frameworks oder Build-Tools eigenmächtig einführen
- Infrastruktur-Annahmen treffen, die nicht explizit gegeben sind
- Secrets oder Credentials im Code hinterlegen
- Container oder Server direkt manipulieren

## Branching & Merge
- Entwicklung auf Feature-/Fix-Branches
- Merge via Pull Request
- main ist geschützt und releasefähig

## Qualitätsanforderungen
- Tests laufen (sofern vorhanden)
- Keine Debug-Ausgaben (console.log, alert, print)
- Keine temporären Workarounds
- Responsive Design intakt (bei Web-Projekten)
- CDN-Links mit Integrity-Attributen (SRI)

## Security
- Keine Secrets im Code oder Repo
- .env niemals committen (stattdessen .env.example)
- Externe Links mit rel="noopener"
- OWASP Top 10 als Mindestcheckliste beachten

## Definition of Done
- Code umgesetzt und getestet
- Doku aktualisiert
- PR reviewed und gemerged
- Deployment erfolgreich

## Design System (optional)
- Farbpalette als CSS-Custom-Properties definieren
- Einheitliche Typografie und Breakpoints festlegen
- Komponentenbibliothek dokumentieren

Eine gute CLAUDE.md spart dir bei jedem Gespräch mit der KI Zeit -- sie muss nicht jedes Mal neu erklärt bekommen, welche Regeln gelten.

Alternative Vibecoding Tools

Claude Code ist nicht das einzige Tool für KI-gestütztes Coding. Hier eine Auswahl weiterer Werkzeuge, die einen Blick wert sind:

  • MiniMax Agent -- KI-Agent, der komplette Web-Anwendungen generieren kann. Einfach Idee beschreiben, und der Agent baut eine fertige App.
  • GitHub Copilot -- KI-Assistent direkt in der IDE (VS Code, JetBrains u. a.). Ergänzt Code in Echtzeit während des Tippens.
  • Cline -- Open-Source KI-Coding-Agent, der als Erweiterung in VS Code läuft. Kann eigenständig Dateien erstellen, bearbeiten und Befehle ausführen.
  • opencode.ai -- Open-Source Terminal-Tool für KI-gestütztes Coding. Leichtgewichtige Alternative, die direkt in der Kommandozeile läuft.

Die Tool-Landschaft entwickelt sich rasant. Probiere verschiedene Werkzeuge aus und finde heraus, welches am besten zu deinem Workflow passt.

Fazit

Vibe-Coding macht Entwicklung zugänglicher, schneller und kreativer.

Es ist besonders wertvoll, um:

  • Ideen schnell zu testen,
  • Konzepte sichtbar zu machen,
  • Anforderungen zu konkretisieren,
  • und mit wenig Einstiegshürde erste digitale Lösungen zu entwerfen.

Wer neugierig ist, klar kommuniziert und bereit ist, iterativ zu arbeiten, kann mit KI erstaunlich weit kommen -- auch ohne klassische Coding-Erfahrung.

Der wichtigste Schritt ist oft nicht Perfektion, sondern einfach der erste Versuch.

Hosting-Anleitung

Schritt-für-Schritt-Anleitung: Vom lokalen Ergebnis bis zur eigenen Website.
Zielgruppe: Workshop-Teilnehmer ohne Vorkenntnisse.

Diese Anleitung zeigt drei Wege, wie du dein Vibe-Coding-Projekt nutzen und zugänglich machen kannst -- vom einfachsten Einstieg bis hin zu fortgeschrittenen Optionen.

Lokale Ergebnisse mit Claude Code

Der einfachste Weg, um mit Vibe-Coding loszulegen: Claude Code liefert dir die fertigen Dateien direkt in einen Ordner auf deinem Computer.

So funktioniert es:

  1. Ordner anlegen -- erstelle auf deinem Desktop (oder an einem beliebigen Ort) einen neuen Ordner, z. B. mein-projekt
  2. Claude Code öffnen -- starte Claude Code und wähle diesen Ordner als Arbeitsverzeichnis
  3. Idee beschreiben -- sage Claude Code, was du bauen möchtest. Die KI erstellt die nötigen Dateien (HTML, CSS, JavaScript) direkt in deinem Ordner
  4. Im Browser öffnen -- öffne die Datei index.html aus deinem Ordner per Doppelklick oder über Datei > Öffnen in deinem Browser

Das ist der schnellste Weg, um deine Ergebnisse zu sehen. Du brauchst keinen Server, kein Konto und keine zusätzliche Software -- nur Claude Code und einen Browser.

Dein Projekt liegt dabei nur auf deinem eigenen Computer. Wenn du es mit anderen teilen oder im Internet zugänglich machen möchtest, lies weiter.

Für Fortgeschrittene: GitHub Pages

Was ist GitHub?

GitHub ist eine Plattform im Internet, auf der man Code-Projekte speichern und mit anderen teilen kann -- ähnlich wie eine Cloud-Ablage, aber speziell für Quellcode. Du brauchst dafür ein kostenloses Konto unter github.com.

Claude Code kann direkt mit GitHub verbunden werden. So werden deine Dateien nicht nur lokal gespeichert, sondern auch auf GitHub hochgeladen. Für kleine, öffentlich zugängliche Projekte bietet GitHub mit GitHub Pages sogar kostenloses Hosting -- deine Seite bekommt eine eigene Internetadresse.

Schritt-für-Schritt-Anleitung

  1. GitHub-Konto erstellen -- registriere dich kostenlos auf github.com
  2. Repository erstellen -- lege ein neues Projekt (Repository) auf GitHub an
  3. Claude Code mit GitHub verbinden -- Claude Code kann deine Dateien direkt in das Repository hochladen
  4. Pages aktivieren -- im Repository unter Settings > Pages den Branch main und Ordner / (root) auswählen, dann speichern
  5. Fertig -- die Seite ist nach ca. 1-2 Minuten unter https://<benutzername>.github.io/<repository-name>/ erreichbar

Änderungen, die auf main hochgeladen werden, werden automatisch veröffentlicht.

Hinweis: Basispfade

GitHub Pages liefert die Seite unter einem Unterpfad aus (z. B. /vibecoding-academy/). Wenn du die Seite lokal öffnest, liegt sie dagegen direkt unter /. Relative Links müssen ggf. angepasst werden.

Self-Hosting mit Docker

Was ist Docker?

Docker ist ein Werkzeug, das eine Anwendung mitsamt allem, was sie braucht, in einen tragbaren „Container“ verpackt. Stell dir das wie einen kleinen, eigenständigen Mini-Computer vor, der deine Website ausliefert. Du installierst Docker einmal auf deinem Rechner, und danach reicht ein einziger Befehl, um deine Seite zu starten.

Docker eignet sich, wenn du deine Seite auf einem eigenen Server betreiben oder lokal in einer professionellen Umgebung testen möchtest.

Voraussetzung

Docker muss installiert sein: https://docs.docker.com/get-docker/

Variante A: Mit Dockerfile (empfohlen für eigene Images)

1. Dockerfile im Projektordner erstellen:

FROM nginx:alpine
COPY . /usr/share/nginx/html
EXPOSE 80

2. Image bauen und starten:

docker build -t vibecoding-academy .
docker run -d -p 8080:80 vibecoding-academy

3. Im Browser öffnen: http://localhost:8080

4. Container stoppen:

docker ps                        # Container-ID nachschauen
docker stop <container-id>       # Container stoppen

Variante B: Ohne Dockerfile (schnelle lokale Vorschau)

Im Projektordner ausführen:

docker run -d -p 8080:80 -v $(pwd):/usr/share/nginx/html:ro nginx:alpine

Dieser Befehl startet einen nginx-Container und bindet den aktuellen Ordner direkt ein -- ohne etwas zu bauen.
