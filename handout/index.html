Worum es in diesem Workshop geht

Die Vibe-Coding Academy soll zeigen, was mit KI-gestuetzter Entwicklung heute schon moeglich ist -- und wie niedrig die Einstiegshuerde dabei sein kann.

Es geht nicht darum, in kurzer Zeit ein vollstaendig ausgereiftes, produktiv einsetzbares Unternehmens-Tool zu bauen. Stattdessen geht es um:

  • Neugier und Ausprobieren
  • Kreativitaet bei der Ideenentwicklung
  • schnelles Prototyping
  • iteratives Verbessern
  • ein besseres Verstaendnis dafuer, wie man mit KI zusammenarbeitet

Vibe-Coding ist kein Hexenwerk. Wer eine Idee beschreiben kann, kann auch erste Ergebnisse erzeugen.

Was Vibe-Coding ausmacht

Vibe-Coding bedeutet, mit einer KI gemeinsam Anwendungen, Prototypen oder Klickdummys zu entwickeln -- oft in natuerlicher Sprache statt ueber klassischen Code von Grund auf.

Der Schluessel ist dabei nicht nur Technik, sondern vor allem:

  • Kreativitaet
  • klare Kommunikation
  • Bereitschaft zur Iteration

Man muss also nicht zwingend selbst programmieren koennen, um etwas zu erschaffen. Hilfreich ist es aber, wenn man:

  • digitale Infrastruktur grob versteht,
  • bestehende Logiken nachvollziehen kann,
  • und bei Bedarf einfachen Code zumindest ansatzweise lesen oder pruefen kann.

Die wichtigste Erkenntnis

Man braucht keine Coding-Erfahrung, um mit Vibe-Coding zu guten ersten Ergebnissen zu kommen.

Was man allerdings braucht:

  • eine Idee,
  • die Faehigkeit, diese Idee zu beschreiben,
  • und die Bereitschaft, der KI Rueckmeldung zu geben.

Zusammenarbeit mit KI: wie mit einem Werkstudenten

Ein hilfreiches Bild ist: Arbeite mit der KI wie mit einem Werkstudenten oder einer Werkstudentin.

Das bedeutet:

  • Anforderungen moeglichst klar formulieren
  • Kontext mitgeben
  • Erwartungen konkret benennen
  • Rueckfragen zulassen
  • Ergebnisse pruefen und nachschaerfen

Je klarer die Aufgabe beschrieben ist, desto besser wird das Ergebnis.

Gleichzeitig darf man nicht erwarten, dass die KI von Anfang an alles perfekt versteht. Gute Ergebnisse entstehen oft im Dialog.

Iterativ statt perfekt von Anfang an

Anders als im klassischen Projekt-, Produkt- oder Anforderungsmanagement muss beim Vibe-Coding nicht jeder Schritt von Anfang an vollstaendig durchpriorisiert und bis ins letzte Detail ausdefiniert sein.

Oft ist es sinnvoller, mit einem ersten Aufschlag zu starten:

  1. Idee grob beschreiben
  2. ersten Prototyp oder ersten Entwurf erzeugen lassen
  3. anschauen, bewerten, testen
  4. Rueckmeldung geben
  5. iterativ verbessern

Typische Rueckmeldungen an die KI sind zum Beispiel:

  • „Den Button wollte ich anders haben.“
  • „Das Dropdown soll sich anders verhalten.“
  • „Hier fehlt noch ein Schritt.“
  • „Der Ablauf ist noch nicht intuitiv genug.“
  • „Bitte ergaenze noch einen alternativen Nutzerpfad.“

Gerade diese iterative Arbeitsweise ist eine der groessten Staerken von Vibe-Coding.

Was Vibe-Coding besonders gut kann

Vibe-Coding ist besonders stark, wenn es darum geht:

  • Ideen schnell sichtbar zu machen
  • theoretische Anforderungen in Prototypen zu uebersetzen
  • Klickdummys fuer Stakeholder zu bauen
  • erste Funktionsideen erlebbar zu machen
  • Anforderungen anschaulich mit anderen abzugleichen

Ein grosser Vorteil: Dinge, die in einem schriftlichen Konzept oft noch abstrakt bleiben, werden im Prototypen viel greifbarer.

Zum Beispiel laesst sich oft erst im Ausprobieren wirklich beurteilen:

  • wie sich ein Button verhalten soll,
  • wie ein Dropdown funktionieren sollte,
  • wie ein Nutzerfluss logisch wirkt,
  • oder welche Informationen an welcher Stelle fehlen.

Nutzen im Projektkontext

Ein vibe-gecodeter Prototyp kann in Projekten sehr wertvoll sein -- auch dann, wenn er nicht produktiv genutzt werden soll.

Er kann helfen als:

  • Gespraechsgrundlage mit Stakeholdern
  • visuelle Ergaenzung zu Fachkonzepten
  • Diskussionsbasis fuer Anforderungen
  • Anschauungsmaterial fuer Entwicklerinnen und Entwickler

So erhalten Entwicklerteams nicht nur ein schriftliches Konzept, sondern auch eine grobe, visuelle Vorstellung davon, wie ein Feature gedacht ist und wie sich Details verhalten koennten.

Das ersetzt keine saubere technische Konzeption -- kann aber das gemeinsame Verstaendnis deutlich verbessern.

Was dieser Workshop ausdruecklich nicht leisten soll

Der Workshop zielt nicht darauf ab, am Ende ein produktionsreifes Tool fuer den Unternehmenseinsatz zu bauen.

Denn dafuer muessten zusaetzlich viele weitere Fragen geklaert werden, zum Beispiel:

  • Datenschutz
  • IT-Sicherheit
  • technische Architektur
  • Wartbarkeit
  • Integration in bestehende Systeme
  • Betrieb und Verantwortlichkeiten
  • Compliance und Governance

Der Fokus des Workshops liegt daher bewusst auf dem Erkunden von Moeglichkeiten -- nicht auf der finalen Produktivsetzung.

Gute Haltung fuer den Einstieg

Wer mit Vibe-Coding starten moechte, braucht vor allem:

  • Neugier
  • Mut zum Ausprobieren
  • Kreativitaet
  • Offenheit fuer Zwischenschritte

Es reicht voellig, mit einer groben Idee zu starten und einfach zu schauen:

„Was passiert, wenn ich diese Anforderung mal an eine KI uebergebe?“

Genau daraus entstehen oft die besten Aha-Momente.

Praktischer Tipp: Idee erst strukturieren lassen

Ein sehr nuetzlicher Ansatz ist, nicht sofort mit dem Coding zu beginnen, sondern die Idee zunaechst mit einer KI zu strukturieren.

Zum Beispiel so:

  • Idee als Sprachnachricht oder freies Brainstorming festhalten
  • einfach 5 bis 10 Minuten „drauflos erzaehlen“
  • die KI bitten, daraus ein strukturiertes Konzept zu machen

Dabei kann die KI helfen, indem sie:

  • Gedanken ordnet
  • Anforderungen strukturiert zusammenfasst
  • offene Punkte identifiziert
  • Rueckfragen zu Missverstaendnissen stellt
  • fehlende Aspekte sichtbar macht
  • eigene Ideen und Ergaenzungen einbringt

So entsteht eine deutlich bessere Ausgangsbasis fuer das eigentliche Vibe-Coding.

Beispiel fuer einen sinnvollen Prompt an eine KI

Ich schildere dir jetzt frei eine Idee fuer ein digitales Tool. Bitte strukturiere meine Gedanken, fasse die Anforderungen uebersichtlich zusammen, markiere offene Punkte und moegliche Missverstaendnisse und stelle mir Rueckfragen. Ergaenze ausserdem eigene Vorschlaege fuer sinnvolle Funktionen oder Aspekte, an die ich vielleicht noch nicht gedacht habe.

Erfolgsfaktoren fuer gutes Vibe-Coding

1. Klar formulieren

Beschreibe Ziel, Nutzergruppe, gewuenschten Ablauf und Funktionsumfang moeglichst konkret.

2. Nicht auf Perfektion beim ersten Versuch warten

Ein erster Entwurf darf unvollstaendig sein.

3. Iterativ arbeiten

Gib Feedback, korrigiere, ergaenze, schaerfe nach.

4. Visualisierung nutzen

Ein Prototyp zeigt oft besser als ein Text, wie etwas gemeint ist.

5. Die KI mitdenken lassen

Bitte die KI aktiv um Rueckfragen, Verbesserungsvorschlaege und alternative Ideen.

6. Ergebnisse realistisch einordnen

Ein guter Prototyp ist noch kein produktionsreifes Produkt.

Fazit

Vibe-Coding macht Entwicklung zugaenglicher, schneller und kreativer.

Es ist besonders wertvoll, um:

  • Ideen schnell zu testen,
  • Konzepte sichtbar zu machen,
  • Anforderungen zu konkretisieren,
  • und mit wenig Einstiegshuerde erste digitale Loesungen zu entwerfen.

Wer neugierig ist, klar kommuniziert und bereit ist, iterativ zu arbeiten, kann mit KI erstaunlich weit kommen -- auch ohne klassische Coding-Erfahrung.

Der wichtigste Schritt ist oft nicht Perfektion, sondern einfach der erste Versuch.

Hosting-Anleitung

Schritt-fuer-Schritt-Anleitung zum Hosten einer statischen Vibecoding-Website.
Zielgruppe: Workshop-Teilnehmer ohne Vorkenntnisse.

Ueberblick

Eigenschaft GitHub Pages Docker Container
Schwierigkeit Sehr einfach Einfach
Kosten Kostenlos Kostenlos
Voraussetzung GitHub-Konto Docker installiert
URL https://<user>.github.io/<repo>/ http://localhost:8080
Geeignet fuer Oeffentliche Seiten Lokale Vorschau, Self-Hosting

Option 1: GitHub Pages

  1. Repository erstellen -- eigenes Repository auf GitHub anlegen oder ein bestehendes forken
  2. HTML-Dateien pushen -- index.html und alle Projektdateien in den main-Branch pushen
  3. Pages aktivieren -- im Repository unter Settings > Pages den Branch main und Ordner / (root) auswaehlen, dann speichern
  4. Fertig -- die Seite ist nach ca. 1-2 Minuten unter https://<benutzername>.github.io/<repository-name>/ erreichbar

Aenderungen, die auf main gepusht werden, werden automatisch veroeffentlicht.

Option 2: Docker Container

Voraussetzung

Docker muss installiert sein: https://docs.docker.com/get-docker/

Variante A: Mit Dockerfile (empfohlen fuer eigene Images)

1. Dockerfile im Projektordner erstellen:

FROM nginx:alpine
COPY . /usr/share/nginx/html
EXPOSE 80

2. Image bauen und starten:

docker build -t vibecoding-academy .
docker run -d -p 8080:80 vibecoding-academy

3. Im Browser oeffnen: http://localhost:8080

4. Container stoppen:

docker ps                        # Container-ID nachschauen
docker stop <container-id>       # Container stoppen

Variante B: Ohne Dockerfile (schnelle lokale Vorschau)

Im Projektordner ausfuehren:

docker run -d -p 8080:80 -v $(pwd):/usr/share/nginx/html:ro nginx:alpine

Dieser Befehl startet einen nginx-Container und bindet den aktuellen Ordner direkt ein -- ohne etwas zu bauen.

Hinweis: Basispfade

GitHub Pages liefert die Seite unter einem Unterpfad aus (z. B. /vibecoding-academy/), Docker dagegen direkt unter /. Wenn relative Links verwendet werden, muessen diese ggf. angepasst werden.

Beispiel:

Kontext Pfad zu einer Unterseite
GitHub Pages /vibecoding-academy/pong/index.html
Docker /pong/index.html
